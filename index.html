Aether : Spatial Behavior Software Suite

  • Edit on GitHub

The Aether: Spatial Behavior Software Suite, is a set of tools and interfaces for designing the behavior of active and responsive environments. Aether also provides tools to facilitate the long-term operation of these spaces.

Use Cases.¶

Developed by the Living Architecture Systems Group and Philip Beesley Studio Inc. Aether was originally created to support Living Architecture Testbeds, and its design has been informed by many years of testing and evolution in the control systems of these deeply immersive environments.

Key Concepts.¶

Distributed Topology:¶

Entities in Aether are designed for distributed computation. This allows the system to be scalable (new devices add minimal computational overhead to the central server), and robust (entities are resilient to network disruption and can continue basic operation without a connection to the server)

Spatial Behavior:¶

Entities in Aether exist in 3D space, and their spatial relationships to their environment and other entities inform their behavior.

Thin Shell Interface:¶

Aether is designed as a Thin Shell, a co-ordinating interface that can speak to many different kinds of hardware and software using open protocols (generally OSC)
