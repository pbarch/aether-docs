Creating an Influence Engine¶

Warning

This documentation page is a work in progress and is not complete.

Introduction to Influence Engines¶

  1. Influence Engines are Algorithms

At the heart of LASG’s behaviour system is the concept of Influence Engines. An influence engine is a piece of software that algorithmically generates ‘influences’ on the various spatially distributed parts of the sculpture. For example, a simple influence engine might create waveforms that move through the sculpture by determining the amplitude of a wave at various physical locations in the space. A different Influence Engine might use particle dynamics to calculate the trajectory of virtual objects which influence elements within the sculpture.

Overview¶

An influence engine needs a name, for this guide the influence engine will be called Ambient Waves, and it will simply create an infinite moving sine wave of influence, with specific angle, period, amplitude and velocity (seen from plan view). This wave will exert an influence proportional to the amplitude of the sine wave in any given location (offset so that its influence is read from 0.0-1.0).

Implement an influence Engine in CPP¶

  1. Copy ExampleEngine.h to AmbientWaves.h and rename all the instances of ExampleEngine to AmbientWaves.

  2. Copy ExampleEngine.cpp to AmbientWaves.cpp and rename all instances of ExampleEngine to AmbientWaves

  3. Add the required header files needed, for ambient waves only math.h is needed 

    #include "math.h"

  4. Add the ambient waves paremeters to the class definition in AmbientWaves.h: 

      float velocity = 0.1;
  float amplitude = 0.25;
  float period = 0.2;
  float angle = M_PI/2;

  5. Register them in populateTable method of AmbientWaves.cpp so they can be sent over OSC and shown in the dashboard 

      tb.add("velocity", &velocity, {.min = 0, .max = 2, .step = 0.01});
  tb.add("angle", &angle, {.min = 0, .max = 2 * M_PI, .step = 0.01});
  tb.add("period", &period, {.min = 0, .max = 1, .step = 0.01});
  tb.add("amplitude", &amplitude, {.min = 0, .max = 1, .step = 0.01});

  6. The influence part of the influence engine is calculate in the influenceOn method

Compiling A WASM Influence Engine¶

  1. Create & compile WASM Influence Engine wrapper

    To recompile an existing WASM Influence engine. Run zig build in the NCv4_Firmware directory. This will build all IE's in the zig.build config file and save the compiled WASMs to the Testbed_Server/public directory.

    To add a new IE to the zig build step, add a buildWasm() call to to the pub fn build(b: *std.Build) function. For example, the ambient waves build function looks like:

    buildWasm(b, "AmbientWaves", &.{
    "AmbientWaves.cpp",
    "AmbientWavesWasm.cpp",
}, optimize);

  2. Register the Influence Engine’s wasm binary in the dashboard
