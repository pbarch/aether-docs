Creating a Logic Reactor Block.¶

Creating a custom logic editor block allows you to add new functionality to logic graphs made with the logic editor.

In this example, we'll create a block that implements both the OR and Exclusive OR logical operators.

Creating a new logic block will involve editing the following files

  1. Adding the new block class in NCv4_Firmware\Blocks.h
  2. Adding our block to the block constructor in NCv4_Firmware\Blocks.cpp
  3. Creating a new module in Testbed_Server\public\logic\modules\[NEW BLOCK].js
  4. Updating the list of modules in Testbed_Server\public\logic\modules.js

Adding a New Block to Blocks.h¶

Block Class Structure¶

We'll start by creating the functionality of the block in NCv4_Firmware\Blocks.h. Each block is defined as a class within this file, and each new block inherits from the parent Block class. A typical block class is structured like this:

class [BlockName] : public Block {
public:

  // declare inputs and their types
  float value = 0.0;

  // declare properties and their types.
  float property = 0.0;

  // declare outputs and their types.
  float outputA = 0.0;
  float outputB = 0.0;

  // Initialize our inputs and outputs
  virtual void init() {
    inputs.addInput("value", &value, {}, 0);
    inputs.addProperty("property", &property, {.min = 0, .max = 1, .step = 0.1});
    outputs.add("outputA", &outputA, {});
    outputs.add("outputB", &outputB, {});
  }

  // define the logic that happens when our block runs
  virtual void execute(unsigned long millis) {
    //logic goes here
  }
};

Variable Declaration¶

For our OR block we'll be looking to create a block with 2 boolean inputs, a parameter that allows the user to set if this is an OR or an Exclusive OR, and a single output.

We'll set up the block initially with the following:

class OrBlock : public Block {
public:

  // declare inputs and their types
  bool inputA = false;
  bool inputB = false;

  // declare properties and their types.
  bool exclusive = false;

  // declare outputs and their types.
  bool output = false;

Initialization¶

Once we've declared our inputs, parameters, and outputs, we need to initialize our block. Initializing adds our blocks and parameters to the Input and Output tables (Tables are a internal data structure used throughout Aether to facilitate memory access and management within WASM's). When we add parameters to the Table we can define some extra options that will determine how the User Interface is will be generated.

Adding an input uses the addInput() method of the inputs table, which looks like:

inputs.addInput("inputA", &inputA, {}, false);

where:
"inputA" is the String Name of our input
&inputA is the memory address of our inputA variable declared above (the & operator returns the memory address of a variable in c++)
{ } is the set of additional options declared for this input, for simple inputs this can typically be left empty
false is the declared baseline for this input (what the input defaults to if nothing is connected)

Adding a property uses the addProperty() of the inputs table which looks like:

inputs.addProperty("property", &property, {.min = 0, .max = 1, .step = 0.1});

This is very similar to the addInput method, but Properties do not have a baseline value defined, and often make use of the additional options argument. Available options that can be defined for a property are:

Type Property Description Default Value
float .min The Minimum value for this property 0
float .max The Maximum value for this property 1.0
float .step The Step size to use for the UI slider (smaller values give finer control) 0.01
bool .display Should this property be displayed in the UI true
char .unit Units to display for this value (mm, cm, m/s etc.) None

Adding an output uses the add() method of the outputs table, which looks like:

outputs.add("outputA", &outputA, {});

where:
"outputA" is the String Name of our input
&outputA is the memory address of our inputA variable declared above (the & operator returns the memory address of a variable in c++)
{ } is the set of additional options declared for this input, for simple inputs this can typically be left empty

Using these Methods we can construct the init() method of our new block.

// Initialize our inputs and outputs
  virtual void init() {
    inputs.addInput("inputA", &inputA, {}, false);
    inputs.addInput("inputB", &inputB, {}, false);
    // We won't use any additional options here as our property is a boolean value
    inputs.addProperty("exclusive", &exclusive, {}); 
    outputs.add("output", &output, {});
  }

Executing Logic¶

With our inputs, properties and outputs initialized we can start writing the logic of our block. The execute function does not return a value, it sets the output variables declared above to the desired value based on the inputs and parameters. (Note the execute function always receives a relative time in milliseconds as an argument, which can be useful for blocks dealing with time, however it is not used in this example)

The execute function for our or block looks like this:

  virtual void execute(unsigned long millis) {
    if (exclusive){
        output = (inputA || inputB) && (inputA != inputB);
    }
    else{
        output = inputA || inputB;
    }
  }

The full class of our OrBlock block to be added to the blocks.h file looks like:

class OrBlock : public Block {
public:

  // declare inputs and their types
  bool inputA = false;
  bool inputB = false;

  // declare properties and their types.
  bool exclusive = false;

  // declare outputs and their types.
  bool output = false;

  // Initialize our inputs and outputs
  virtual void init() {
    inputs.addInput("inputA", &inputA, {}, false);
    inputs.addInput("inputB", &inputB, {}, false);
    // We won't use any additional options here as our property is a boolean value
    inputs.addProperty("exclusive", &exclusive, {}); 
    outputs.add("output", &output, {});
  }

  // Execute our logic
  virtual void execute(unsigned long millis) {
    if (exclusive){
        output = (inputA || inputB) && (inputA != inputB);
    }
    else{
        output = inputA || inputB;
    }
  }
};

Updating the Block Constructor in Blocks.cpp¶

Now that we've created our block class, we need to add it to the Block constructor defined in NCv4Firmware\Blocks.cpp.

The block constructor starts around line 13 and begins with Block *newBlock(const char *type) { followed by a long list of conditional statements. To the end of this list of conditionals, we need to add the condition of our new block. We'll do this by adding another else if condition at the end of the list before the errorLog(" Can't find %s in Blocks.cpp .... ", type); warning. Our new condition will look like:

else if (strcmp(type, "OrBlock") == 0){
    return new OrBlock();
}

This completes the creation of the block on the firmware side of the Aether Code base.

Recompile WASM's and Firmware.¶

With the firmware side complete we can rebuild the Web Assembly Modules from the NCv4_Firmware folder by running zig build in the terminal. We can also upload firmware with our new block to our devices via OTA or directly over usb. In the next section we'll create the files necessary to include our new block in the Logic Editor UI.

Creating a new module¶

To make our new block display in the User Interface of the Logic Reactor we need to create a new module that defines how the block will be drawn. This module will be added to Testbed_Server\public\logic\modules\ and should match the class name of the block defined in Blocks.h.

for this example we'll create OrBlock.js and add it to the modules folder. The following template can be used to create a new block module:

import { html } from "../../global.js"
import { Position } from "../../libraries/xyflow.js"
import { NodeContainer } from "../components.js"
import { useErrorSignal } from "../../components/hooks.js";
import { millis } from "../shared.js"

export const inputs = {

}

export const outputs = {

}

export function initWasm(data) {
}

export function execute({}, data) {
  data.wasm.executeBlock(millis());
}

export function Display({data}) {
  return html`
    <${NodeContainer} data=${data}>
      <div class="text-center" style="height:30px; width:40px;">
        <!-- YOUR BLOCK DISPLAY INFO -->
      </div>
    <//>
  `
}

export function DisplayIcon() {
  return html`
    <${NodeContainer}>
      <div class="text-center" style="height:30px; width:40px;">
        <!-- YOUR BLOCK DISPLAY INFO -->
      </div>
    <//>
  `
}

The inputs and outputs constants need to be populated with the inputs and outputs of our block. Each input and output must be added as a Javascript Object, with the following properties:

inputA: {
    type: "number",
    description: "First value",
    position: Position.Left,
  },
Property Description
type The Javascript type of this input (*in general this should always be "number")
desciription A description of this input
position determines where the input will appear on the block. Options are Position.Left, Position.Right, Position.Top, Position.Bottom. Note we generally use Left or Top for inputs, with the outputs directly across from the input (Right or Bottom)

Note

The spelling of the input or output must match exactly with the String Name of the input or output that was defined in the initialization function of the block class.

The complete input and output constants for our OR block would look like:

export const inputs = {
  inputA: {
    type: "number",
    description: "First Value",
    position: Position.Left,
  },
  inputB: {
    type: "number",
    description: "Second Value",
    position: Position.Left,
  },
}

export const outputs = {
  output: {
    type: "number",
    description: "Result",
    position: Position.Right,
  },
}

The Display() and DisplayIcon() functions allow us to describe how a block will look in the Logic Editor using Javascript and HTML. Replacing the <!-- YOUR BLOCK DISPLAY INFO --> Text in the template with text will create a simple text label for the block.

In the Display() function we also have access to data about inputs, outputs and parameters of the block. In our example, we'll use this to change our simple text label depending on the 'exclusive' property of our block:

export function Display({data}) {
  const exclusive_ref = useErrorSignal(data.data.settings, "exclusive")
  let label_text = "OR"
  if(exclusive_ref.value){label_text = "XOR"}

  return html`
    <${NodeContainer} data=${data}>
      <div class="text-center" style="height:30px; width:40px;">
        <!-- YOUR BLOCK DISPLAY INFO -->
        ${label_text}
      </div>
    <//>
  `
}

We use const exclusive_ref = useErrorSignal(data.data.settings, "exclusive") call to get a reference to the "exclusive" property of our block. We can check the value of this reference using exclusive_ref.value and use this to change the text that appears in the block label.

The full code of our OrBlock.js file is:

Updating Modules.js¶

Finally we need to update the Testbed_Server\public\logic\modules.js file to include our new module. Open the file and add the following to the imports at the top of the file:

import * as OrBlock from "./modules/OrBlock.js";

We also need to add our imported OrBlock to the export const modules = { list below.

Once this has been added our new Logic Block is complete!
