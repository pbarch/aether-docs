Compiling NC Firmware¶

Setting up the compilation buildchain.¶

  1. Install Platformio Cli python get-platformio.py in NCv4_Firmware directory
  2. Install Zig
  3. In the NCv4_Firmware directory run zig build arduino to compile the firmware. The compiled binary can be found at NCv4_Firmware/build/NCv4_Firmware.ino.bin

Over The Air Update¶

  1. In the NCv4_Firmware directory run zig build arduino to create the OTA bins
  2. On the Testbed Dashbaord select the node controller you wish to update
  3. In the Hardware tab select OTA Update to send the new NC.bin to all selected NC's

Warning

For major firmware changes test the update on one NC before uploading to all NC's.

Issues in major changes can lead to the NC being stuck in a boot loop, and in these cases working firmware will need to be reuploaded over a usb connection. Doing this for a large number of NC's is a hassle.

Upload over USB¶

  1. Attach the NC to your PC via USB
  2. in the NCv4_Firmware directory run pio device list to see if the NC is discovered and see it's port
  3. run pio run -t upload -e [ENV] where [ENV] is the env from platformio.ino for the plugged in board

Adding a board configuration¶

  1. Use pio boards list [search term] to find the correct board name
  2. Add an environment to platformio.ini where board is the name from 1
  3. Add build flags
    • OTA_SUFFIX string for differentiating OTA binaries (required)
    • other c macro defintions can be used to control compilation
  4. Add board to build.zig in the array named boards (search for const boards =) using the format .{<board name>, <OTA_SUFFIX>},
